    Contact methods¶

    What you can do with Contact methods¶

    The Fulfil API lets you do the following with the Contact method resource. More detailed versions of these general actions may be available:

    The Contact method object¶

    Attributes¶

    • comment text

      Comment

      Comments. Usually used for notes like "Don't call after 5"

    • id integer

      Readonly

      ID

      Unique ID of the record

    • name string

      Name

      Optional name of the contact or department this method is associated to

    • type string

      Type Supported choices are:

      • phone
      • mobile
      • fax
      • email
      • website
      • skype
      • sip
      • irc
      • jabber
      • other
      • twitter
      • facebook
      • instagram

    • value string

      Value

    More attributes¶

    View less frequently used attributes

    • active boolean

      Active

      Set to false to archive a contact method

      The active field's boolean value indicates if Contact method is archived. See archive and restore to archive and restore records.

    • attachments array of integer ids

      Attachments

      List of documents attached to the record

    • create_date timestamp

      Readonly

      Created At (Timestamp)

      UTC timestamp when the record was created

    • create_uid integer id

      Readonly

      Create User

      ID of the user that created the record.

    • email string

      E-Mail

    • facebook string

      Facebook

    • instagram string

      Instagram

    • metadata json

      Metadata

      A JSON field that can be used to store custom data

    • other_value string

      Value

    • sequence integer

      Sequence

    • sip string

      SIP

    • skype string

      Skype

    • twitter string

      Twitter

    • url string

      URL

    • website string

      Website

    • write_date timestamp

      Readonly

      Updated At (Timestamp)

      UTC timestamp when the record was last updated. Coule be null

    • write_uid integer id

      Readonly

      Write User

      ID of the user that last updated the record

    Retrieve a specific Contact method¶

    GET /model/party.contact_mechanism/{id}

    model = client.model('party.contact_mechanism')
record = model.get(record_id)

    Retrieve a list of Contact methods.¶

    Retrieve all Contact methods

    GET /model/party.contact_mechanism

    Supported URL parameters¶

    Parameter Description
    since_id Show Contact methods with an id greater than given id
    ids Retrieve only Contact methods specified by a comma-separated list of order IDs.
    created_at_min Show Contact methods created at or after date (format: 2020-12-25T16:15:47) UTC date times
    created_at_max Show Contact methods created at or before date (format: 2020-12-25T16:15:47) UTC date times
    updated_at_min Show Contact methods updated at or after date (format: 2020-12-25T16:15:47) UTC date times
    updated_at_max Show Contact methods updated at or before date (format: 2020-12-25T16:15:47) UTC date times
    per_page The maximum number of results to show on a page.
    page The page of results to display

    Fetches the second page of results.

    GET /model/party.contact_mechanism?page=2

    To fetch the next page, pass page as a parameter to the request.

    GET /model/party.contact_mechanism?page=2&per_page=100

    Also fetches the second page, but with a 100 results per page. The maximum records allowed is 500 per page.

    GET /model/party.contact_mechanism?since_id=12345

    Retrieve a list of Contact methods after a specified ID. Restricts results to those after the specified ID.

    GET /model/party.contact_mechanism?updated_at_min=2019-04-17T19:42:24

    Retrieve all Contact methods changed after a certain time Pass an UTC datetime (formatted as ISO string).

    GET /model/party.contact_mechanism?ids=12345,12346

    Retrieve a specific list of Contact methods Restrict results to Contact methods specified by a comma-separated list of IDs.

    Retrieve a count of Contact methods.¶

    GET /model/party.contact_mechanism/count

    Returns a specific Contact method

    Create Contact method¶

    POST /model/party.contact_mechanism

[
    {
        "comment": "Lorem Ipsum is simply dummy text of the printing and typesetting industry. Lorem Ipsum has been the industry\u0027s standard dummy text ever since the 1500s,",
        "name": "String",
        "type": "instagram",
        "value": "String"
    }
]

    model = client.model('party.contact_mechanism')
value_list = [{'comment': "Lorem Ipsum is simply dummy text of the printing and typesetting industry. Lorem Ipsum has been the industry's standard dummy text ever since the 1500s,", 'name': 'String', 'type': 'instagram', 'value': 'String'}]
new_record_ids = model.create(value_list)

    Common gotchas:

    • The create endpoint always takes a list of objects (this is intentional).
    • The returned response is the list of ids of the created Contact methods.

    Example response

    [42]

    Update Contact method¶

    PUT /model/party.contact_mechanism/{id}

{
    "comment": "Lorem Ipsum is simply dummy text of the printing and typesetting industry. Lorem Ipsum has been the industry\u0027s standard dummy text ever since the 1500s,"
}

    from fulfil_client import Client

client = Client('merchant_id', 'API_KEY')

model = client.model('party.contact_mechanism')
changes = {'comment': "Lorem Ipsum is simply dummy text of the printing and typesetting industry. Lorem Ipsum has been the industry's standard dummy text ever since the 1500s,"}
model.write([42, 43], changes)

    Response

    {
    "comment": "Lorem Ipsum is simply dummy text of the printing and typesetting industry. Lorem Ipsum has been the industry\u0027s standard dummy text ever since the 1500s,",
    "id": 42
}

    Delete Contact method¶

    DELETE /model/party.contact_mechanism/#{id}

    model = client.model('party.contact_mechanism')
model.delete([42, 43])

    Response

    Status: 204 No Content

    Archive Contact method¶

    PUT /model/party.contact_mechanism/#{id}/archive

    model = client.model('party.contact_mechanism')
model.archive([42, 43])

    Restore Contact method¶

    PUT  /model/party.contact_mechanism/#{id}/restore

    model = client.model('party.contact_mechanism')
model.restore([42, 43])

    Advanced search & filter¶

    To learn more about the supported search expressions, read Searching & Filtering Guide

    To get a list of ids that match the filter, send a PUT request to

    PUT  /model/party.contact_mechanism/search

{}

    If you're only interested in getting a count, you can call the :code:search_count endpoint instead.

    PUT  /model/party.contact_mechanism/search_count

{}

    You can also avoid two API calls (a search and then a read) by using the :code:search_read endpoint.

    PUT  /model/party.contact_mechanism/search_read

{}
